Graph Algorithms · Lesson 03

Depth-First Search

An algorithm that dives deep into a graph before backtracking — exploring every path to its end before trying another.

01. What is DFS?

Depth-First Search traverses a graph by going as far as possible down one branch before backtracking. Think of it like exploring a maze: you follow one path until you hit a dead end, then backtrack to the last fork and try another route.

DFS can be implemented with recursion (using the call stack implicitly) or an explicit stack data structure. Both approaches have identical behavior.

02. How it Works

01
Start at Source Pick a starting node and mark it as visited. Add it to the call stack.
02
Explore Neighbors Look at the current node's unvisited neighbors in adjacency list order.
03
Recurse Call DFS on the first unvisited neighbor — go as deep as possible.
04
Backtrack When all neighbors are visited (or none exist), return to the previous node.
05
Terminate Repeat until all reachable nodes have been visited.

03. Pseudocode

procedure DFS(graph, node, visited)
    // Mark this node so we don't revisit it
  mark node as visited
  
  for each neighbor of node:
    if neighbor is not visited:
      DFS(graph, neighbor, visited)  // recurse deeper
    end if
  end for
    // All neighbors done → backtrack automatically
end procedure

04. Python Implementation

def dfs(graph, node, visited=None):
    if visited is None:
        visited = set()            # initialize on first call
    
    visited.add(node)             # mark as visited
    print(node, end=" ")

    for neighbor in graph.get(node, []):
        if neighbor not in visited:
            dfs(graph, neighbor, visited)

# --- Example graph (adjacency list) ---
graph = {
    'A': ['B', 'C'],
    'B': ['D', 'E'],
    'C': ['F'],
    'D': [],
    'E': ['F'],
    'F': []
}

dfs(graph, 'A')
# Output: A B D E F C

05. Complexity Analysis

Case Time Space Explanation
General O(V + E) O(V) Visit each vertex once; traverse each edge once. Stack depth = max O(V).
Dense Graph O(V²) O(V) With adjacency matrix where E ≈ V², time becomes O(V²).
Tree (no cycles) O(N) O(H) N = nodes, H = tree height (stack depth equals height).
Practice Exercises
Q1 Trace the DFS traversal for the example graph starting at 'A'. Why might the order change? ▼

Walk through the graph manually or use the visualizer above. Think about: what happens at each node? When does it backtrack? How does the order of neighbors in the adjacency list affect the result?

Trace: A → visits B (first neighbor) → B visits D (first neighbor of B, dead end) → B visits E → E visits F (F is unvisited here) → backtrack to E, B, A → A visits C → C would visit F but it's already visited.

Output: A B D E F C

Why order matters: DFS explores neighbors in the order they appear in the adjacency list. If 'C' was listed before 'B' in A's neighbors, the traversal would go A → C → F → B → D → E (F already visited). The topology is the same graph, but the traversal order depends entirely on adjacency list ordering.
Q2 What is the time and space complexity of DFS? ▼

Consider both the number of vertices (V) and edges (E). For space, think about what's stored on the call stack in the worst case.

Time: O(V + E) — Each vertex is visited exactly once (O(V)), and for each vertex we look at all its edges (O(E) total across all vertices).

Space: O(V) — The visited set is O(V). The recursion stack depth is at most O(V) in the worst case (a linear chain graph). For iterative DFS with an explicit stack, the stack can also hold up to O(V) elements.
Q3 Name three real-world applications of DFS. ▼

Think broadly — DFS is used in compilers, games, networking, and more. Consider what problems "exploring as deep as possible" naturally solves.

1. Maze solving — DFS follows one path to completion before trying alternatives, mirroring how humans trace a maze.

2. Topological sorting — Used in compilers to resolve build dependencies. DFS produces a post-order that can be reversed to get a valid topological order.

3. Detecting cycles in graphs — DFS tracks nodes on the current stack; if a back-edge to a stack node is found, a cycle exists. Critical in dependency management (detecting circular imports).

Bonus: Finding strongly connected components (Tarjan's/Kosaraju's algorithms), solving puzzles like Sudoku, web crawling, and parsing expressions in compilers.
Q4 How would you modify DFS to detect if a graph contains a cycle? ▼

This is a common interview question. Think about what additional information you'd need to track during the traversal.

Track two sets: visited (all ever-visited nodes) and rec_stack (nodes on the current recursion path).

During DFS, add each node to rec_stack on entry and remove it on exit. If you encounter a neighbor that's already in rec_stack, you've found a back edge → cycle detected.

A node in visited but not in rec_stack is safe (it was fully explored via another path). This runs in O(V + E) — no extra cost over standard DFS.
